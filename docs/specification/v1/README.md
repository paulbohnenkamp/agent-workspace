# Agent Workspace Platform Specification v1.0 RC

## Status

Release candidate for the Agent Workspace Platform specification.

## Principles

1. **Workspace-centric** — work happens inside durable collaboration containers.
2. **Artifact-centric** — the durable result of work is an artifact, not a chat log.
3. **Human + agent collaboration** — humans and agents participate through shared workspace state.
4. **Metadata-driven composition** — workspace experiences are assembled from definitions.
5. **Definition/runtime separation** — definitions describe work; instances execute work.
6. **Declarative workspace definitions** — new workspace types are definitions, not new applications.
7. **Durable state and versioning** — artifacts, definitions, runs, and actions are persistable.
8. **Extensible through packages** — workspace definitions, playbooks, policies, and components can be packaged.

## Normative Language

- **MUST** indicates an absolute requirement.
- **SHOULD** indicates a recommended behavior.
- **MAY** indicates an optional behavior.
- **MUST NOT** indicates prohibited behavior.
