# First-Class Object Specifications

Every first-class object SHOULD define:

- purpose
- responsibilities
- schema
- relationships
- lifecycle
- persistence rules
- events
- versioning rules
- examples
- security considerations

## Objects

- WorkspaceDefinition
- WorkspaceInstance
- WorkItem
- ArtifactDefinition
- ArtifactInstance
- KnowledgeSource
- Action
- Thread
- Run
- PlaybookDefinition
- PlaybookInstance
- AgentDefinition
- AgentSession
- SkillDefinition
- ToolDefinition
- Participant
- Event
