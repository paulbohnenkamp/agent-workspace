# Workspace Object-Panel Mapping

This document provides a manual object-to-panel mapping for the current workspace originals.

Use it when annotating the PNGs by hand. It is intentionally descriptive rather than coordinate-based so the mapping stays useful even if the visuals evolve.

## Shared Reading Pattern

Across the current workspace set:

1. the far-left column is the `Work Queue`
2. the center-left column is the `AI Assistant`
3. the center-right region is the primary `Output` plus supporting outputs
4. the right rail is ordered as `Knowledge Sources`, `Agents (x)`, then `Actions`
5. the header above the main body is the active `Work Item`

## Annotation Guidance

When annotating by hand:

1. mark the `Work Item` header first
2. mark the `AI Assistant` and primary `Output` next because they explain the core interaction flow
3. keep the entire right rail grouped as context plus control, but label each panel separately
4. use `Agents` rather than `Agent Activity` so the wording stays aligned with the current visual system
5. label `Supporting Output` only where the lower center-right panel is clearly distinct from the primary output
